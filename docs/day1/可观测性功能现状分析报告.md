## 4. Java 层实现组件

### 4.1 TokenUsageTracker — LLM Token/成本追踪（`@Component`）

**文件**: `common/observability/TokenUsageTracker.java`

核心 Token 追踪引擎，通过 `TokenTrackingAdvisor` 驱动：

| 指标 | 类型 | 标签 | 说明 |
|:---|:---|:---|:---|
| `deepresearch.llm.calls.total` | Counter | agent, tier | LLM 调用次数 |
| `deepresearch.llm.tokens.input` | Counter | agent, tier | 输入 Token 数 |
| `deepresearch.llm.tokens.output` | Counter | agent, tier | 输出 Token 数 |
| `deepresearch.llm.cost.estimated` | Counter | agent, tier | API 成本（微美元，Counter 不支持浮点） |
| `deepresearch.llm.latency` | Timer | agent, tier | 调用延迟（含百分位 SLO） |
| `deepresearch.llm.fallback.total` | Counter | agent, from, to | 模型降级次数 |

DeepSeek V4 成本估算价格（2026年7月参考）：

| 模型 | 输入 / 1M tokens | 输出 / 1M tokens |
|:---|:---|:---|
| Pro | $0.55 | $2.19 |
| Flash | $0.14 | $0.55 |

额外功能：`trackFallback()`（降级事件）、`getSessionCost()`（会话累计成本）、`clearSession()`（会话结束清理）。

### 4.2 TokenTrackingAdvisor — 透明拦截器（order 2100）

**文件**: `common/observability/TokenTrackingAdvisor.java`

Spring AI `BaseAdvisor`（优先级 `LOWEST_PRECEDENCE + 100`），装配于 `AgentBundle` 和 `EnterpriseChatClientConfig` 的 Advisor 链末尾：

- **before()**: 记录 `System.currentTimeMillis()` 到 advisor context
- **after()**: 提取 `ChatResponse.usage` 元数据 → 计算延迟 → 调用 `tokenTracker.track()`

Agent 身份（`agent`, `tier`, `sessionId`）通过 `.advisors(a -> a.param(...))` 传入。

**Advisor 链**（AgentBundle，5 个）:
```
TokenBudget[200] → PiiMask[300] → OutputGuard[300] → TokenTrack[900] → AuditLog[100]
```

## 11. 设计亮点

| 特性 | 实现方式 | 说明 |
|:---|:---|:---|
| **Tracer 可选注入** | `ObjectProvider<Tracer>` | OTel SDK 未配置时降级为 UUID traceId，不影响启动 |
| **双路指标采集** | 直抓 + OTLP 推送 | Prometheus 始终可用，OTLP 开启后支持 Jaeger |
| **零侵入 Token 追踪** | Advisor 透明拦截 | 所有 ChatClient 调用自动采集，无需修改 Agent 代码 |
| **成本微美元存储** | Counter × 1,000,000 | 避免浮点 Counter，Grafana 面板除以 1,000,000 |
| **按租户 Eval 隔���** | `ConcurrentHashMap<tenant, AtomicRef>` | 修复全局单值 Gauge 多租户互相覆盖 |
| **Spring AI 内置指标** | `management.observations.enable.spring.ai: true` | ChatClient/Tool/VectorStore 指标自动暴露 |
| **Resilience4j 自动集成** | `resilience4j-micrometer` 依赖 | CB 状态/失败率自动暴露，无需手动注册 |
| **节点级 Timer** | WorkflowTracingHelper | 7 个工作流节点独立的 duration Timer |

## 12. 已知限制

| 限制 | 影响 | 说明 |
|:---|:---|:---|
| Jaeger 需 OTLP 配置 | 分布式追踪默认不可用 | 需设置 `OTEL_METRICS_ENABLED=true` + OTel Collector |
| `@Timed`/`@Observed` 未使用 | AOP 基础设施闲置 | 切面 Bean 已注册，但业务代码未使用注解式埋点 |
| 无日志聚合 | 日志分散在本地文件 | 未接入 Loki/ELK，需登录服务器查看 |
| 无告警通知渠道 | 告警仅在 Prometheus UI 可见 | 未配置 AlertManager → Slack/飞书 webhook |
| 指标无长期存储 | 15 天留存后滚动删除 | 生产环境建议接入 VictoriaMetrics/Thanos |
